## 👨‍💼 Инструкция для мастера

> Всё что нужно знать, чтобы пользоваться сайтом каждый день. Никаких технических знаний не требуется.

---

### С чего начать

Сайт состоит из **двух частей**:

| Часть | Для кого | Что там |
|-------|----------|---------|
| **Публичный сайт** | Клиенты | Смотрят услуги, записываются онлайн |
| **Панель мастера** | Только ты | Управление записями, клиентами, финансами, складом |

---

### 📅 Шаг 1 — Открой рабочие дни

> Клиент **не сможет записаться**, пока ты не откроешь день в календаре.

**Как это сделать:**
1. Зайди в панель мастера → раздел **«Расписание»** или **«Даты»**
2. Нажми на нужный день → **«Открыть»**
3. По желанию укажи часы работы (например, 10:00 – 19:00)

> Если день нужно закрыть (заболел, выходной) — нажми **«Закрыть день»**. Новых записей на этот день уже не будет.

---

### ✂️ Шаг 2 — Настрой услуги

> Клиент видит список услуг на сайте и выбирает нужную перед записью.

**Как добавить или изменить услугу:**
1. Панель мастера → **«Услуги»**
2. Нажми **«Добавить»** или нажми на существующую
3. Заполни: название, цену, длительность (в минутах), описание
4. По желанию добавь фотографии работ

> Длительность услуги важна — она влияет на то, какие временные слоты предлагаются клиенту. Например, если стрижка 90 минут, то слот в 18:30 при окончании работы в 19:00 показан **не будет**.

---

### 📲 Шаг 3 — Подключи Telegram-уведомления

> Когда клиент записывается — ты мгновенно получаешь сообщение в Telegram.

**Что нужно сделать один раз:**
1. Напиши [@BotFather](https://t.me/BotFather) в Telegram → `/newbot` → создай бота
2. Скопируй **токен бота** (выглядит как `1234567890:ABCdef...`)
3. Узнай свой **Chat ID** — напиши боту [@userinfobot](https://t.me/userinfobot), он скажет твой ID
4. Передай эти два значения разработчику или вставь в настройки сервера:
   - `TELEGRAM_BOT_TOKEN` = токен бота
   - `TELEGRAM_MASTER_CHAT_ID` = твой Chat ID

**После этого** ты будешь получать уведомления о каждой новой записи, переносе, отмене и завершении визита.

---

### 🔔 Напоминания клиентам

> За 24 часа до визита клиент автоматически получает напоминание в Telegram — но только если он **подписался на бота**.

**Как клиенту подписаться:**
1. После записи на сайте появляется кнопка/ссылка на бота
2. Клиент нажимает **«/start»** в боте
3. Всё — теперь бот знает этого клиента и пришлёт напоминание

> Если клиент не подписался — напоминание не отправится. Это нормально.

---

### 📋 Как работать с записями каждый день

#### Новая запись пришла сама (онлайн)
1. Ты получил уведомление в Telegram 📋
2. Запись автоматически появилась в расписании со статусом **«Активная»**
3. Ничего делать не нужно — просто жди клиента

#### Клиент пришёл и ты принял его
1. Найди запись в расписании (по дате)
2. Нажми **«Завершить»**
3. Укажи итоговую цену, чаевые (если были), аренду кресла (если есть)
4. Нажми **«Сохранить»**

> При завершении система автоматически спишет расходники со склада и посчитает себестоимость.

#### Клиент опаздывает
1. Найди его запись
2. Нажми **«Опоздание»**
3. Укажи на сколько минут опаздывает
4. Можно сдвинуть время начала — тогда система проверит, не конфликтует ли это с другими записями

#### Перенос записи
1. Открой запись → измени дату и/или время
2. Статус автоматически изменится на **«Перенесён»**
3. Мастер получит уведомление с указанием «было/стало»

#### Отмена записи
1. Открой запись → нажми **«Отменить»**
2. Слот освободится — другой клиент сможет на него записаться

### 💰 Финансы

> Все деньги считаются автоматически — тебе нужно только правильно завершать записи.

**Как посмотреть отчёт:**
1. Панель мастера → **«Финансы»**
2. Выбери период (например, текущий месяц)
3. Увидишь:

| Показатель | Что это |
|-----------|---------|
| **Выручка** | Сумма всех оплат за услуги |
| **Чаевые** | Сумма всех чаевых |
| **Аренда** | Сколько заплатил за аренду кресла |
| **Себестоимость** | Расход на краски и материалы |
| **Прибыль** | Выручка + Чаевые − Аренда − Расходники |

> Себестоимость рассчитывается **автоматически** на основе склада расходников. Если склад не заполнен — себестоимость будет 0.

---

### 📦 Расходники (склад)

> Здесь хранится информация о красках, материалах и их остатках.

**Как добавить расходник:**
1. Панель мастера → **«Расходники»** → **«Добавить»**
2. Заполни поля:
   - **Тип**: краска (`paint`) или материал (`material`)
   - **Бренд и название**: например, Wella / Koleston 7/0
   - **Цвет**: для красок (например, «натуральный блонд»)
   - **Остаток**: сколько сейчас есть (в граммах или штуках)
   - **Минимальный запас**: при каком остатке предупреждать о нехватке
   - **Объём упаковки** + **Стоимость упаковки**: нужно для расчёта цены за грамм

**Как пополнить запас:**
- Найди расходник → нажми **«Пополнить»** → укажи количество

**Поиск расходника:**
- В строке поиска введи любое слово — бренд, название или цвет

**Инвентаризация:**
- Кнопка **«Инвентаризация»** покажет:
  - Все позиции с текущими остатками
  - Количество позиций с низким запасом (красная метка ⚠️)
  - Общую стоимость склада в рублях

---

### 👥 База клиентов

> Клиенты добавляются автоматически при первой записи. Ты можешь добавить к ним заметку.

**Как добавить комментарий к клиенту:**
1. Панель мастера → **«Клиенты»**
2. Найди клиента → нажми **«Редактировать»**
3. Напиши комментарий (например: «аллергия на аммиак», «любит длинную стрижку»)

> Этот комментарий будет отображаться в Telegram-уведомлениях, когда клиент записывается снова.

---

### ❓ Частые вопросы

**Клиент хочет записаться на нестандартное время — что делать?**
> На сайте есть кнопка **«По договорённости»**. Клиент указывает желаемые дни, а тебе приходит запрос в Telegram. Ты договариваешься напрямую и потом вручную создаёшь запись в панели.

**Клиент не может выбрать дату — почему?**
> Скорее всего, день не открыт. Зайди в **«Даты»** и открой нужные дни.

**Слот занят, но там никого нет — что делать?**
> Найди «пустую» запись и либо **удали** её, либо смени статус на **«Отменена»** — слот освободится.

**Как записать клиента самому (без онлайн-формы)?**
> В панели мастера есть кнопка **«Добавить запись вручную»** — заполни форму так же, как заполнял бы клиент.


## 🧩 Модули и функционал

### 1. Онлайн-запись (публичная часть)

- Отображение услуг с ценами, длительностью и фотографиями
- Выбор даты из открытых мастером рабочих дней
- **Умный выбор слотов**: система рассчитывает свободные окна с шагом 30 мин, учитывая длительность выбранной услуги и уже занятые интервалы
- Вариант **«по договорённости»**: клиент указывает желаемые дни, мастер получает запрос в Telegram
- Форма: имя, Telegram и/или телефон
- **Проверка конфликтов**: система возвращает HTTP 409 + данные конфликтующей записи при пересечении времени

### 2. Расписание (панель мастера)

- Просмотр записей на день / диапазон дат
- Управление рабочими днями: открыть / закрыть день, задать часы работы (WorkStart–WorkEnd)
- Изменение статуса записей
- Фиксация опоздания (`late_min`) с возможностью сдвинуть время начала
- Добавление комментария к записи

### 3. Финансы

```
Прибыль = Выручка + Чаевые − Аренда − Себестоимость расходников
```

- Отчёт за произвольный период
- Поля на каждой записи: `price`, `tips`, `rent`, `supply_cost`
- `supply_cost` рассчитывается **автоматически при завершении** записи и фиксируется исторически (не меняется при изменении цен позже)

### 4. База клиентов

- Автоматическое создание при первой записи
- Поиск по Telegram / телефону
- Комментарий мастера к клиенту (отображается в уведомлениях)
- История визитов (через фильтр по контакту)

### 5. Услуги

- CRUD-управление услугами
- Поля: название, длительность, цена, категория, описание, фотографии (JSON-массив URL)
- Шаблон расходников для услуги (`service_supplies`) — автозаполнение при завершении

### 6. Управление расходниками (склад)

> Подробнее в разделе [Расходники и склад](#-расходники-и-склад)

```
│ comment       │ text                                 │
│ reminder_sent │ bool                                 │
│ created_at    │ timestamp                            │
└───────────────┴─────────────────────────────────────┘
```

```
│  · Опоздание                     │    │  · Ставит reminder_sent=true  │
│  · Запрос на индив. время        │    │                               │
└──────────────────────────────────┘    └───────────────────────────────┘
```

## 📦 Расходники и склад

### Логика работы

```
ДОБАВЛЕНИЕ РАСХОДНИКА
─────────────────────
  • type: paint (краска) или material (материал)
  • При добавлении закупки: quantity_grams + total_cost
    → автоматически вычисляется cost_per_unit = total_cost / quantity_grams
  • min_quantity — порог: если quantity ≤ min_quantity → low_stock = true

ПРИМЕНЕНИЕ К ЗАКАЗУ
────────────────────
  Вариант А (шаблон услуги):
    service_supplies → автоматически при завершении

  Вариант Б (ручной список):
    supplies_used = JSON: [{supply_id: 3, quantity: 50}, ...]
    Перекрывает шаблон услуги

  При смене статуса → completed:
    1. computeSupplyCost() → apt.supply_cost (фиксируется навсегда)
    2. deductSupplies()    → GREATEST(quantity - N, 0) в БД (атомарно)

ИНВЕНТАРИЗАЦИЯ (GET /api/supplies/inventory)
──────────────────────────────────────────────
  {
    "items":             [...все расходники с вычисленными полями],
    "low_stock_count":   3,
    "total_stock_value": 12450.0   // сумма (quantity × cost_per_unit)
  }

ПОПОЛНЕНИЕ (POST /api/supplies/:id/restock)
────────────────────────────────────────────
  Body: { "quantity": 500 }
  → quantity += 500 (атомарная операция в БД)

ПОИСК (GET /api/supplies/search?q=wella)
──────────────────────────────────────────
  Поиск по: brand ILIKE, name ILIKE, color ILIKE
  Отсортировано: type ASC, brand ASC, name ASC
```

### Схема связей услуги и расходников

```
Service ──── service_supplies ──── Supply
  id              service_id         id
  name            supply_id          brand
  ...             quantity           name
                                     quantity (остаток)
                                     cost_per_unit
```

## 💰 Финансы

### Формула прибыли

```
Прибыль = Выручка + Чаевые − Аренда − Себестоимость расходников
profit   = total_revenue + total_tips − total_rent − total_supply_cost
```

### Важно: историческая себестоимость

`supply_cost` у записи **фиксируется в момент завершения** и больше не меняется.
Это значит: даже если цена расходника изменится завтра — старые отчёты останутся корректными.

```
Завершение записи:
  ┌─────────────────────────────────────────────────┐
  │ 1. Определить список расходников:               │
  │    supplies_used JSON → или шаблон услуги        │
  │                                                 │
  │ 2. Для каждого расходника:                      │
  │    cost = total_cost / quantity_grams × qty     │
  │                                                 │
  │ 3. apt.supply_cost = сумма (округление 0.5)     │
  │                                                 │
  │ 4. Сохранить запись в БД                        │
  │                                                 │
  │ 5. Списать остатки со склада (GREATEST(q-n, 0)) │
  └─────────────────────────────────────────────────┘
```
